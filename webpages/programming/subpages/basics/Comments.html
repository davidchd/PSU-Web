Comments 注释

Comments is a very essential part of internal documentation in programming. Almost every known programming language will provide some kind of comments features. In C++, there are two kinds of comments: single-line comments and multiple-line comments.
注释是编写任何程序都很重要的参考文档。几乎任何已知的程序语言都会提供某种形式的注释功能。在C++中，注释分为两种类型：单行注释和多行注释(多行注释有时又称块注释或段注释)。在上面的两个代码块中，第一块是两个单行注释的例子，第二块是两个多行注释的例子。

Here are two examples of single-line comment. 这是两个单行注释的例子。 


// this is a single-line comment

int n = 10; // this is also a single-line comment

And here are two examples of multiple-line comment. 这是两个个多行注释的例子 


/* this is a multiple-line comment */

/*
    this is also a multiple-line comment
*/

The point of having comments is to increase the readability of a source code file. Comments are usually constructed with explanatory words or sentences, in order for other programmers who might be reading this program to understand the logic flow. Comments may also help the person who wrote this program to locate any specific part in the source code file much easier.
注释的主要目的是增加程序可读性。程序的注释一般由解释性的词或句子组成。这些内容旨在帮助其他可能阅读该程序的程序员更好地理解程序逻辑。注释也可以帮助编写这个程序的程序员更容易的找到程序中的特定部分。

Single-line Comment 单行注释

A single-line comment will start with two slashes //. When a C++ program is being compiled, the compiler will ignore any content after two slashes until it has reached the end of this line. Technically, a single-line comment can be written at anywhere in a program as long as it does not interfere with executable code. However, there are some common ways of using single-line comments:
当C++程序被编译的时候，编译器会自动忽略任何从双斜杠开始到该行行尾的内容。理论上，只要不影响到程序指令，单行注释可以写在程序的任何位置。但是，我还是会举出一些常见的使用单行注释的方式。

Comment can be used to indicate the start of a part of the program. 注释可以用来表示程序中一部分的开始。 


// input part
cin >> a;

// output part
cout << a + 10;

Comment can be used to note a statement when is used after a command. 注释用在一行命令后面时，可以表示对这行指令的备注。 


int sum; // the sum of all numbers

Comment can be used to show the use of a repetition or selection statement when being used after a closing curly bracket }. 注释用在回大括号后面时，可以解释大括号所对应的循环或选择语句的作用。 


if(num < 0) {
    cout << "Negative";
} else {
    cout << "Nonnegative";
} // output negative if num is less than zero

Multiple-line Comment 多行注释

There are two common ways of using a multiple-line comment.
多行注释有两个常见的主要用法。

The first way is to write at the very beginning of a program, which will be a comment section for this source code file. Such a comment section should include author's name, program created date, source file name, etc.
第一种用法是写在程序的最开头。写在这种位置的注释一般表示整个程序的注释，应该包括程序作者名，程序创建日期，源代码文件名，等。 


/*
    Source file: main.cpp
    Written by David Chen.
    Created on Nov 5th, 2018.
    Last modified on Nov 7th, 2018.
*/
#include <iostream>
using namespace std;
...

The second way of using a multiple-line comment is to temporarily disable a block of unfinished code when debugging. This allows you to test your program even though some part of it can not be compiled yet.
多行注释的第二个常见用法是在测试程序时注释掉一部分未完成的代码。这可以让你注释掉无法编译的部分并随时测试程序。 


cin >> a;
/*
if() {
    a += 10;
}
*/
cout << a;
